1. PRODUCT REQUIREMENTS DOCUMENT (PRD)
1.1 Purpose

Build a portfolio website that demonstrates:

Engineering depth

System thinking

Ownership of full stack projects

Measurable impact and reasoning

The site should convince a hiring engineer in under 90 seconds that:
“This candidate can build and reason about real systems.”

1.2 Target Audience

Primary:

Software engineers

Backend engineers

Hiring managers

Technical recruiters

Secondary:

Internship coordinators

Hackathon evaluators

1.3 Core Goals

Clearly communicate technical direction

Showcase projects with engineering depth

Demonstrate understanding of performance, tradeoffs, and architecture

Be fast, stable, and simple

1.4 Key Features
A. Landing Page

One line technical positioning statement
Example:
“Full Stack Engineer with strong backend and systems fundamentals.”

Quick access to:

Projects

Resume

GitHub

LinkedIn

B. Projects Section (Most Important)

Each project must include:

Problem statement

Architecture overview

Tech stack and why it was chosen

Key engineering challenges

Performance metrics

What you would improve next

Projects to highlight:

Job Portal

WhatsApp Clone

DB Storage and I/O Research

C. Skills Section

Group skills by domain, not tools

Programming

Backend and APIs

Databases

Core CS

Tooling

D. Resume Section

Downloadable PDF

ATS friendly version

Highlight backend and system work

E. About Section

Focus on:

How you think

What problems you enjoy

How you learn

Avoid personal fluff.

F. Contact Section

Email

LinkedIn

GitHub

No contact forms. Engineers email.

1.5 Non Functional Requirements

Page load under 2 seconds

Lighthouse score above 90

Mobile responsive

SEO friendly

Zero broken links
